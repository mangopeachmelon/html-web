# ⚽ 유럽 5대 축구 리그 정보 웹사이트 (European Top 5 Leagues)

> **"축구를 사랑하는 대학생들이 모여 만든, 유럽 축구 입문자를 위한 가이드 웹사이트"**

유럽을 대표하는 5대 축구 리그(프리미어리그, 라리가, 세리에 A, 분데스리가, 리그 앙)의 특징과 주요 팀, 핵심 선수를 소개하는 반응형 웹사이트입니다. 복잡한 정보 검색 없이도 한눈에 팀의 정보를 파악할 수 있도록 직관적인 카드 디자인으로 제작했습니다.

---

## 📅 프로젝트 개요
* **프로젝트 기간:** 2025년 12월 1일(월) ~ 2025년 12월 13일(토) (약 2주)
* **참여 인원:** 3명 (팀 프로젝트)
* **프로젝트 목표:**
    1.  HTML/CSS의 기초를 활용하여 웹사이트의 전체적인 구조를 설계한다.
    2.  팀원들과 협업하여 30개 이상의 페이지를 통일감 있게 제작한다.
    3.  반응형 웹을 적용하여 모바일에서도 깨지지 않는 화면을 구현한다.

## 🛠️ 사용 기술 (Tech Stack)

### **Frontend**
* **HTML5:**
    * 웹 표준을 준수하기 위해 시맨틱 태그(`header`, `nav`, `main`, `section`, `footer`)를 적극적으로 사용했습니다.
* **CSS3:**
    * **Layout:** `Flexbox`와 `Grid` 시스템을 적절히 섞어 카드 레이아웃을 깔끔하게 잡았습니다.
    * **Design:** `Glassmorphism` (유리 질감) 효과를 헤더에 적용하여 최신 트렌드를 반영했습니다.
    * **Responsive:** `@media` 쿼리를 사용하여 스마트폰(모바일) 환경에서도 사이드바가 작동하도록 구현했습니다.
* **JavaScript (Vanilla JS):**
    * `Event Listener`를 사용하여 햄버거 메뉴 클릭 시 사이드바가 부드럽게 나오는 기능을 구현했습니다.
    * 페이지 이동 시 `Fade-in/out` 효과를 넣어 뚝 끊기는 느낌 없이 자연스럽게 넘어가도록 만들었습니다.

## ✨ 주요 기능 및 특징

### 1. 직관적인 네비게이션 시스템
* **어디서든 이동 가능:** 어떤 페이지에 있더라도 상단 헤더(로고)와 사이드바를 통해 다른 리그나 메인 화면으로 쉽게 이동할 수 있습니다.
* **모바일 최적화:** 모바일 환경에서는 화면을 가리지 않도록 햄버거 메뉴(`☰`)를 통해 공간을 효율적으로 활용했습니다.

### 2. 통일감 있는 디자인 시스템 (Design System)
* **폰트:** 가독성이 좋은 `Pretendard` 폰트를 사용하여 한글과 영문 모두 깔끔하게 보이도록 했습니다.
* **카드 UI:** 모든 팀 리스트를 카드 형태로 만들어 마우스를 올리면(Hover) 이미지가 확대되고 떠오르는 효과를 주어 클릭을 유도했습니다.
* **일관된 구조:** 30개 팀 페이지 모두 **[팀 소개 - 주요 선수 - 네비게이션]**의 통일된 구조를 적용하여 사용자가 정보를 예측하기 쉽게 만들었습니다.

### 3. 디테일한 시각 효과
* **배경 패턴:** 단순한 단색 배경 대신 은은한 도트(점) 패턴을 넣어 심미성을 높였습니다.
* **선수 프로필:** 선수 사진을 원형으로 가공하고 그림자와 테두리를 주어 입체감을 살렸습니다.
* **그라데이션:** 주요 타이틀에 브랜드 컬러 그라데이션을 적용하여 강조했습니다.

## 💡 프로젝트 후기 & 배운 점 (Retrospective)

* **협업과 파일 경로:** 팀원끼리 각자 맡은 리그 폴더를 작업하다 보니 이미지 경로(`../img/`)를 맞추는 과정에서 실수가 잦았습니다. 초반에 **폴더 구조와 파일명 규칙(Naming Convention)을 정하는 것**이 얼마나 중요한지 깨달았습니다.
* **CSS의 재사용성:** 처음에는 페이지마다 `<style>` 태그를 썼지만, 나중에 `style.css` 하나로 통합하면서 **유지보수의 편리함**을 몸소 체험했습니다. 클래스명만 잘 지어두면 디자인을 한 번에 바꿀 수 있다는 점이 놀라웠습니다.
* **반응형 웹의 원리:** PC 화면뿐만 아니라 모바일 화면까지 고려하여 `px` 대신 `%`나 `rem` 단위를 고민하고, 미디어 쿼리를 작성하는 과정에서 CSS 실력이 많이 늘었습니다.
